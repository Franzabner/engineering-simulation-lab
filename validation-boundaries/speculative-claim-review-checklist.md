# Speculative Claim Review Checklist

Status: scaffolded
Publication state: public scaffold
Release state: not released

## Purpose

This checklist prevents speculative engineering notes from becoming unsupported claims.

## Claim Gates

| Claim type | Public-safe requirement |
| --- | --- |
| Hypothesis | Clearly framed as a question or study category. |
| Simulation | Labeled scaffolded unless separately reviewed. |
| Benchmark | No benchmark result or output is included. |
| Result | No measured result is included. |
| Proof | No proof-completion language is included. |
| Deployment | No deployment or production-readiness language is included. |

## Held Out

Do not include breakthrough claims, measured outputs, raw traces, validated simulation claims, proprietary methods, customer claims, deployment claims, production-readiness claims, or proof-completion claims.

## Escalation

If wording implies validation, breakthrough, or production use, rewrite as no-result hypothesis framing or hold for review.
